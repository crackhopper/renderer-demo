# PowerShell Shell Guidance

## Overview

This project uses **PowerShell** as the default shell on Windows. All shell commands generated by the AI assistant SHALL default to PowerShell syntax.

## Shell Command Conventions

### Directory Listing

| Bash | PowerShell |
|------|------------|
| `ls` | `Get-ChildItem` or `dir` |
| `ls -la` | `Get-ChildItem -Force` |
| `ls *.txt` | `Get-ChildItem -Filter "*.txt"` |

### File Operations

| Bash | PowerShell |
|------|------------|
| `cp src dst` | `Copy-Item -Path src -Destination dst` |
| `mv src dst` | `Move-Item -Path src -Destination dst` |
| `rm file` | `Remove-Item -Path file` |
| `mkdir -p dir` | `New-Item -ItemType Directory -Path dir` |

### Path Handling

- Use Windows paths: `C:\Users\...` not `/c/Users/...`
- PowerShell allows both forward `/` and back `\` slashes in paths
- Environment variables: `$env:VAR_NAME` instead of `$VAR`

### Glob Patterns

| Bash | PowerShell |
|------|------------|
| `*.txt` | `-Filter "*.txt"` with `Get-ChildItem` |
| `**/*.txt` | `Get-ChildItem -Recurse -Filter "*.txt"` |
| `find . -name "*.txt"` | `Get-ChildItem -Recurse -Filter "*.txt"` |

### Command Chaining

| Bash | PowerShell |
|------|------------|
| `cmd1 && cmd2` | `cmd1; cmd2` or `cmd1 & cmd2` |
| `cmd1 \|\| cmd2` | `if ($?) { cmd2 }` or `cmd1; if (-not $?) { cmd2 }` |
| `cmd > file` | `cmd > file` (works in PowerShell) |

### Silent Operation

| Bash | PowerShell |
|------|------------|
| `cmd > /dev/null 2>&1` | `cmd > $null 2>&1` |

## Implementation

The AI assistant SHALL:
1. Use `Get-ChildItem`, `Copy-Item`, `Move-Item`, `Remove-Item` cmdlets for file operations
2. Use `-Filter` parameter for glob-style pattern matching
3. Use Windows path conventions
4. Use `$null` instead of `/dev/null` for output suppression
5. Use `&` or `;` for command chaining

## Exceptions

Only use bash syntax if:
- The user explicitly requests bash commands
- The command targets a WSL/Linux subsystem
- A specific tool only supports bash (e.g., certain git hooks)
