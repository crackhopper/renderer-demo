## Context

The AI assistant system operates in a Windows environment using Cursor IDE with PowerShell as the default shell. However, the default system prompts and training assume a bash/Linux environment, causing a mismatch between generated commands and the actual execution environment.

## Goals / Non-Goals

**Goals:**
- Ensure all shell commands generated by the AI use PowerShell syntax
- Provide clear guidance on PowerShell equivalents for common bash operations
- Eliminate need for users to translate bash commands to PowerShell

**Non-Goals:**
- Modifying the underlying AI model or language model
- Changing Cursor IDE's shell configuration
- Supporting multiple shell environments (focused on PowerShell only)

## Decisions

### 1: Where to Add PowerShell Guidance

**Decision:** Add PowerShell caveats to the system prompt/agent instructions.

**Rationale:** The system prompt is the most direct way to influence AI behavior without code changes. The `.cursor/rules/` or AGENTS.md directory is the standard location for Cursor project-specific guidance.

**Alternatives Considered:**
- Creating a separate PowerShell helper script: Too indirect, AI still generates bash by default
- Modifying skill files: Only affects specific commands, not general shell behavior

### 2: Content of PowerShell Caveats

**Decision:** Include the following in the system prompt:
- Explicit statement that PowerShell is the default shell
- Common PowerShell equivalents for bash syntax (e.g., `ls` → `Get-ChildItem` or `dir`, glob patterns with `-Filter`)
- Path format conventions (Windows paths vs Unix paths)
- Command chaining syntax differences (`;` works in both, but `$null` for silent operation)

**Rationale:** Concrete examples are more effective than abstract instructions for ensuring correct command generation.

## Risks / Trade-offs

- **Risk:** Overly verbose system prompt could reduce AI effectiveness on other tasks
  - **Mitigation:** Keep caveats concise and focused on shell command generation

- **Risk:** Some bash idioms have no clean PowerShell equivalent
  - **Mitigation:** Provide clear "no direct equivalent" guidance with suggested alternatives
